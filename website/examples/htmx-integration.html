Macro meets Micro: DOM Swapping with Cleanup

This example demonstrates how front.js components handle DOM thrashing from macro-frameworks like HTMX without memory leaks.

How It Works

The Problem: When HTMX (or similar libraries) swap DOM content, components with timers, event listeners, or other side effects can leak memory if not cleaned up.

The Solution: front.js attaches a _front_dispose function to each island container. Before HTMX swaps content, we call this function to cleanup all reactive runs and their side effects. 

// Global HTMX cleanup handler
document.body.addEventListener('htmx:beforeCleanup', (event) => {
  if (event.target._front_dispose) {
    event.target._front_dispose();
  }
});
